# WSL 环境风险诊断报告

生成时间：2026-04-25

---

## 风险一：`.venv_x5` 是空壳，命名具有误导性

**现象：**
- `.venv_x5` 存在，Python 版本为 3.12.3
- 内部**没有安装任何 horizon/hmct/hbdk 工具链包**
- `run_x5_quant.sh` 中 `VENV_DIR` 指向的是 `.venv_oe_py310`，不是 `.venv_x5`

**原因分析：**
- `.venv_x5` 是最初尝试用系统 Python 3.12 创建的虚拟环境
- Horizon open_explorer v1.2.8 只支持 Python 3.10，Python 3.12 无法安装工具链包
- 安装失败后改用 `.venv_oe_py310`（Python 3.10），`.venv_x5` 被遗弃但未删除
- 根目录存在的 `=0.2.4`、`=1.7` 等垃圾文件，正是在 `.venv_x5` 中尝试 pip install 时命令行写错产生的

**风险：**
- 新接手人员看到 `.venv_x5` 可能误以为这是正确的工具链环境
- 如果误执行 `source .venv_x5/bin/activate` 后运行量化，会立即报错 `hb_mapper not found`

---

## 风险二：Python 环境存在三套，版本混乱

**现象：**

| 环境 | Python | 工具链 | 状态 |
|------|--------|--------|------|
| 系统 `/usr/bin/python3` | 3.12.3 | 无 | 系统级，不可用于量化 |
| `.venv_x5` | 3.12.3 | 无 | 空壳，废弃 |
| `.venv_oe_py310` | 3.10 | v1.2.8（当前主力） | ✅ 可用 |
| `.venv_oe_py310_v123` | 3.10 | v1.2.3（旧版） | 备用，版本低 |

**风险：**
- 没有任何文档或标记说明哪个是"正确"环境
- `run_x5_quant.sh` 硬编码了 `.venv_oe_py310`，但如果有人手动执行 hb_mapper 而不用脚本，很容易用错环境
- 两套 Python 3.10 环境（v1.2.8 和 v1.2.3）的 hbdk 版本不同（3.49.15 vs 3.49.9），同一个 ONNX 在两个版本下量化结果可能不同，无法保证一致性

---

## 风险三：多个 YAML 配置文件中存在失效引用

**现象：**

| YAML | 问题 |
|------|------|
| `x5_quant_best_nv12.yaml` | `cal_data_dir` 指向 `calibration_bin_float32_nchw`，该目录**不存在** |
| `x5_quant_best_rgb.yaml` | 同上，`calibration_bin_float32_nchw` 不存在 |
| `x5_quant_nv12_fixed.yaml` | `onnx_model` 指向 `best_fixed.onnx`，该文件**不存在** |

**风险：**
- 3 个 YAML 中有 2 个引用了不存在的 calibration 目录，1 个引用了不存在的 ONNX
- 直接运行这些配置会报错，但错误信息不直观，排查耗时
- `run_x5_quant.sh` 默认使用 `x5_quant_best_nv12.yaml`（车辆模型），该配置的 cal 目录不存在，**直接运行脚本不带参数会失败**

---

## 风险四（附加）：两套 open_explorer 版本共存，量化结果不可追溯

**现象：**
- `open_explorer_x5/`（v1.2.8）和 `open_explorer_x5_v123/`（v1.2.3）同时存在
- `x5_quant_output_v123/` 目录为空，说明 v1.2.3 量化失败或未完成
- 现有 `.bin` 文件均由 v1.2.8 生成，但没有任何文件记录使用了哪个版本

**风险：**
- 无法从 `.bin` 文件本身判断是哪个工具链版本生成的
- 如果未来需要重新量化，必须确保使用相同版本，否则结果不可复现
- 两个压缩包（共 3.65 GB）占用大量磁盘空间，且已解压，原始 tar.gz 可考虑归档

---

## 总结：最大风险点（Top 3）

| 优先级 | 风险 | 影响 |
|--------|------|------|
| 🔴 高 | `.venv_x5` 空壳误导 + 默认 YAML 的 cal 目录不存在，直接运行脚本会失败 | 新人无法复现量化流程 |
| 🟠 中 | 3 个 YAML 配置存在失效引用（不存在的文件/目录） | 误用配置导致量化失败，排查困难 |
| 🟡 低 | 两套工具链版本共存，无版本标记，量化结果不可追溯 | 未来重新量化时结果可能不一致 |
